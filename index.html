Known gotchas

  • NomadStar firmware bundles ship with a hardcoded PKI keypair — every unit has the same pub/priv key out of the box. Confirmed by seeing 2+ nodes in the area with identical publicKey. Fix: --factory-reset-device via admin message; CLI bug requires sending the int directly, not bool — see reset_keys.py.
  • mqtt.enabled silently held false on nRF52 builds with no WiFi until mqtt.proxy_to_client_enabled is also true. Set both together.
  • nRF52 nodes need an external MQTT gateway (phone app, web client, or PC bridge). Phone app on Android is unreliable in background. Web client can't reach the broker (no WSS). See mqtt_bridge.py for the native-TCP fallback if you want a deterministic test path.

Jim's Nodes — Field Setup

📟 LILYGO T-Deck Plus EXT (house + mobile)

This is a standalone Meshtastic computer — no phone needed. Setup is done on the device itself using its touchscreen + keyboard. Most important things:

  1. Set owner: Settings → Device → set Long Name (e.g. "Jim T-Deck") and Short Name (4 chars max)
  2. Set region: Settings → LoRa → Region → US
  3. Set role: Settings → Device → Role → CLIENT (not Router — this device moves)
  4. Enable WiFi (at home): Settings → Network → WiFi → SSID + password → enables direct MQTT gateway
  5. Enable MQTT: Settings → Module → MQTT → on → broker mqtt.meshtastic.org / user meshdev / pwd large4cats
  6. Enable GPS: Settings → Position → GPS Mode → Enabled, smart broadcast on
  7. OK to MQTT: Settings → LoRa → Config OK to MQTT → on
  8. Import FarleyFam: scan the QR code or paste the shared channel URL (above) — uses the camera or USB connection from a phone
  9. Channel 0 uplink: Settings → Channels → 0 → Uplink Enabled + Downlink Enabled

Once configured at home, his T-Deck becomes a gateway for the whole local NH mesh too — a real contribution to the community.

📦 RAK WisBlock Starter Kit (Merrymeeting cabin)

Same architecture as your Repeater Mini: nRF52840, no WiFi. Setup via USB to a PC:

  1. python reset_keys.py (adjust COM port) — unique PKI keys
  2. Edit farls-danvers-solar-optimized.yaml — change owner (e.g. "Jim Lake Cabin"), owner_short (e.g. "JLC"), lat/lon (cabin coords), alt
  3. meshtastic --port COMx --configure farls-danvers-solar-optimized.yaml
  4. meshtastic --port COMx --ch-index 0 --ch-set uplink_enabled true --ch-set downlink_enabled true
  5. Add FarleyFam: meshtastic --port COMx --ch-add FarleyFam --ch-index 1 --ch-set psk <the_psk> --ch-set uplink_enabled true --ch-set downlink_enabled true (use the PSK from your dashboard ⚙ Settings)
  6. python node_health.py COMx — verify 7-point checklist
